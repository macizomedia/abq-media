# Draft Continuation Plan

## Phase A — Release Readiness (Alpha)
1. Add root npm scripts for easier usage:
   - `yt:doctor`, `yt:prep`, `yt:latest`
2. Add `.env` support and prioritize env vars over file secrets.
3. Improve error reporting in `prep` to show which fallback failed and why (compact trace).
4. Add smoke tests:
   - transcript-file success
   - text input success
   - latest command success
   - provider fallback behavior
5. Publish as `0.1.0-alpha.1` with npm tag `alpha`.

## Phase B — Transcription Hardening
1. Add timeout controls for yt-dlp and ASR uploads.
2. Add chunking for long audio uploads (if API limits hit).
3. Add a `--max-duration-min` guard.
4. Add cached transcript reuse by video ID.

## Phase C — Podcast MVP
1. `podcast` command implementation:
   - generate Spanish script from deep research markdown
   - TTS render to mp3
   - write episode metadata
2. Add SoundCloud upload adapter.
3. Add publish dry-run mode.

## Phase D — Video Engine (Later)
1. Audiogram template rendering with ffmpeg.
2. Subtitle burn-in option.
3. YouTube upload adapter.

## Publish Checklist (Alpha)
1. Ensure package metadata fields are final:
   - name, version, description, repository, homepage, bugs, license
2. Ensure files included in package (`files` field).
3. `npm whoami` and org access check (`@abquanta`).
4. Version bump prerelease:
   - `npm version 0.1.0-alpha.1 --no-git-tag-version`
5. Publish:
   - `npm publish --access public --tag alpha`
6. Tag + changelog + README examples.

## Security Follow-up
- Rotate exposed API keys.
- Move secrets to env vars.
- Add `.abq-module.json` to `.gitignore` where appropriate.
